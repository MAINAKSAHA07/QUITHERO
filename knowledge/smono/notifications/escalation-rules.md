# Escalation Protocol

## Level 1: Craving Spike (≥2 in 2 hours)

1. Send craving-spike notification immediately
2. If user opens app within 30 min → direct to breathing exercise
3. If no response after 2 hours → send gentle follow-up (different message)
4. Max 2 craving-spike notifications per day

## Level 2: Missed Session (2+ consecutive days)

1. Day 1 missed: Standard missed_session notification
2. Day 2 missed: More urgent — "We noticed you haven't been back. Everything OK?"
3. Day 3 missed: Reduce ask — "Even 2 minutes today. Just a mood check."
4. Day 4+ missed: Back off completely — 1 notification every 3 days max

## Level 3: Slip Event

1. Immediate (within 5 min): Compassionate, zero-guilt message
2. +6 hours: "Day [X] has a section on exactly this. Ready when you are."
3. +24 hours: If no session started, send Day 25 content teaser (slip recovery day)
4. Never send more than 3 slip-related notifications total per event

## Level 4: Intensity Trend Rising (3+ days)

1. First detection: Adjust next scheduled notification to grounding-focused
2. Second day: Add an extra craving-support notification at peak hour
3. Third day: Fast-track relevant content (stress toolkit if stress-reactor, etc.)

## De-escalation

When signals improve (intensity falling, sessions resuming, mood improving):
- Return to normal notification frequency within 48 hours
- Send positive reinforcement: "Noticed things settling? That's real progress."

## Hard Rules

- NEVER use guilt or shame in escalation messages
- NEVER imply failure or disappointment
- ALWAYS frame slip as normal part of process
- ALWAYS offer a low-effort re-entry point (mood check, breathing exercise)
- Night silence (11 PM – 7 AM) applies even for escalation (except immediate slip)
